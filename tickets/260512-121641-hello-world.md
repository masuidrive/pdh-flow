# hello-world — `pdh-flow hello` subcommand

## Why

A tiny, end-to-end-shaped coding ticket: smallest possible change that
touches the CLI, a new module, and a test — useful for dogfooding the
flow / the Web UI's run view without doing anything risky.

## What

Add a `hello` subcommand to the pdh-flow CLI:

- `pdh-flow hello` → prints `hello, world` to stdout, exits 0
- `pdh-flow hello --name <name>` → prints `hello, <name>`
- `pdh-flow hello --name ""` (empty) → falls back to `hello, world`
- Listed in the CLI usage text (`pdh-flow help` / `pdh-flow` with no args)

Implementation: new `src/cli/hello.ts` exporting the handler; wire it into
the dispatch in `src/cli/index.ts`; add it to the usage block.

## Acceptance Criteria

- AC-1: `pdh-flow hello` writes exactly `hello, world\n` to stdout and exits 0. *(unit-test-sufficient)*
- AC-2: `pdh-flow hello --name Yuichiro` writes exactly `hello, Yuichiro\n` and exits 0. *(unit-test-sufficient)*
- AC-3: `pdh-flow hello --name ""` writes `hello, world\n` (empty name → "world"). *(unit-test-sufficient)*
- AC-4: `pdh-flow` (no args) and `pdh-flow help` list `hello` in the subcommands. *(unit-test-sufficient)*

## Out of scope

- i18n / multiple greetings
- Reading the name from stdin or a config file
- Colored output
